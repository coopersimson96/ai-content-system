# Graphic Type Decision Tree

For every slide in `02-slide-copy.md`, pick exactly one graphic type. Don't blend types — that's how slides start looking inconsistent.

## The 8 types

### 1. Proof card
**Use when:** the slide references a real source — article, brand, doc, video, screenshot, official page.
**Looks like:** a bordered evidence panel containing the source asset, with a label strip (`EVIDENCE 01`, `SOURCE`, or `PROOF`) and a tiny caption.

### 2. Prompt box
**Use when:** the slide teaches a prompt structure or shows exact instructions.
**Looks like:** a dark or white code-block card with monospace text. Highlight key tokens (`ROLE:`, `EXAMPLES:`, `DON'TS:`) in the accent color.

### 3. Before / after panel
**Use when:** the slide compares weak vs strong (bad prompt → good prompt; broken → fixed).
**Looks like:** two stacked cards side-by-side or top-bottom. Left/top = the weak version (greyed). Right/bottom = the strong version (highlighted in accent).

### 4. Stack diagram
**Use when:** the slide explains a formula, framework, or layered system.
**Looks like:** stacked rectangles or `+` symbols connecting labelled blocks. e.g. `Role + Examples + Don'ts + Task + Output Format`.

### 5. Checklist / card grid
**Use when:** the slide summarizes 3–5 discrete tips or steps.
**Looks like:** a 2×2 or 1×N grid of equal cards, each with a small label and one short sentence. Optional accent on the most important card.

### 6. Evidence teardown
**Use when:** the slide critiques a real artifact (email, ad, website, UI).
**Looks like:** the screenshot pinned right with a thin accent border, big diagnosis headline left, and numbered overlay callouts on the screenshot (`#1`, `#2`).

### 7. CTA poster
**Use when:** it's the final slide.
**Looks like:** big headline, the CTA keyword in a button or pill (`Comment "PROMPT"`), supporting "I'll DM you the breakdown" line, save/share buttons, mascot/footer signature.

### 8. Oversized typography poster
**Use when:** the slide carries an abstract insight or punchline that needs no diagram.
**Looks like:** the headline IS the slide. Type fills 70%+ of the canvas, broken across multiple lines, with one keyword in the accent color. Minimal supporting copy.

## Quick map: slide role → likely graphic type

| Slide role | Default graphic type |
|---|---|
| Hook (slide 1) | Oversized typography poster |
| Source / proof | Proof card or Evidence teardown |
| Mistake / problem | Oversized typography poster or Before/after |
| Sauce / non-obvious | Oversized typography or Prompt box |
| Rule / point | Oversized typography or Prompt box |
| Formula / system | Stack diagram |
| Checklist / steps | Checklist grid |
| What's missing | Oversized typography |
| CTA (final slide) | CTA poster |

## Hard rule

Don't put two graphic types on the same slide. One slide = one job.
