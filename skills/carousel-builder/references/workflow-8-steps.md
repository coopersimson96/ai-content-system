# The Reference-First Carousel Workflow

The non-negotiable rules behind why this skill exists.

## Why most AI carousels look bad

People open one chat, type "make me a 6-slide carousel about X," hit enter, and get back 6 slides that don't feel like the same carousel. Different fonts. Different vibes. Different layouts.

The model isn't the problem. The brief is.

## The 8 steps

1. **Pick one sharp idea** — not a topic, a thesis. "AI prompting" is a topic. "Anthropic's playbook says strong prompts are role + examples + don'ts" is a thesis.
2. **Write the slide copy first** — every slide gets a headline + support line + optional bottom note before any image is generated.
3. **Collect 2–4 references** — IG carousels, posters, book covers, brand graphics. Two = thin. Four = enough variety. More than four = the model loses the signal.
4. **Tell the model what to *borrow* from each reference** — not "make it like this." Assign each reference a job: typography, layout, colour/mood, pacing.
5. **Generate 3 versions of slide 1** — same copy, three visually distinct cover concepts.
6. **Refine the best one** — that locked slide becomes the visual anchor for the entire carousel.
7. **Generate the rest, one slide at a time** — every later slide references the anchor and matches typography, spacing, colour, texture, mood, hierarchy.
8. **Check consistency, regenerate the weak ones, post** — never regenerate everything; only the weak slides with specific notes.

## The borrow / do not copy split

For every reference image, spell it out:

```
Borrow:
- typography hierarchy
- spacing
- colour treatment
- texture
- visual pacing
- layout logic
- graphic components

Do not copy:
- exact text
- exact branding
- exact graphics
- exact compositions
- creator identity
```

This single distinction changes the output completely. References stop being "make it look like this" and start being "borrow these specific things."

## Refinement language that works

When refining the anchor or any weak slide:

- **"Make the headline larger and more dominant."**
- **"Reduce the clutter and keep one strong focal point."**
- **"Make the typography feel less generic, more premium."**
- **"Keep the headline. Change the background. Don't touch the footer."**

Always tell it what to **keep**, what to **change**, and what to **not touch**. Otherwise it'll change everything and you lose the parts that were working.

## Default 6-slide structure

```
Slide 1: Hook        — the headline that stops the scroll
Slide 2: Mistake     — what most people do wrong
Slide 3: Sauce       — the non-obvious unlock
Slide 4: Anchor rule — the one rule that holds it all together
Slide 5: Formula     — the repeatable system in one line
Slide 6: CTA         — what to do next
```

## Default 8-slide structure

```
Slide 1: Hook
Slide 2: Source / proof / context
Slide 3: Point 1
Slide 4: Point 2
Slide 5: Point 3
Slide 6: Formula / framework
Slide 7: What's missing / expanded resource
Slide 8: CTA
```
